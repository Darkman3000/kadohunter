# Project Rules & Standards

## Documentation Storage

- Rule: All project-specific documentation must be stored in `todo/`.
- Reasoning: Keeps plans, operating notes, and handoff records versioned with the codebase.

## Documentation Discipline

- Update decision docs when a meaningful product or architecture choice changes.
- Update progress tracking in the same pass as implementation slices.
- Do not store live secrets in project documentation.
- Keep templates reusable; keep live project facts in the active project docs.
